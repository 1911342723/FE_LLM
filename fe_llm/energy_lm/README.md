# energy_lm —— 最小能量对话模型（M6）

用第一性哲学"能量从不稳定走向稳定"训出的、能生成连贯语言的最小模型。
**非自回归、非 softmax 接龙、能量驱动、可溯源**。

## 跑起来

```bash
python -m fe_llm.energy_lm.train        # 训练（去掩码=刻低能深沟）
python -m fe_llm.energy_lm.demo         # 你好→你好 演示 + 可溯源 + 省电曲线
python -m fe_llm.energy_lm.ablation_per # 退化验证：注意力是 PER 的特例
```

## 文件

| 文件 | 作用 |
|---|---|
| `corpus.py` | 最小对话语料（85 对，"你好→你好"级别）= 经验 |
| `tokenizer.py` | 字级分词器（含 [MASK]/[BOS]/[EOS]/[SEP]） |
| `energy_net.py` | 能量网络 E_θ + **PER 交互机制**（自有，非注意力） |
| `train.py` | 去掩码训练：让真实回应能量低 = 刻低能深沟 |
| `collapse.py` | 能量坍缩生成：全 [MASK] → 逐位置填最低能字 → 回应浮现 |
| `demo.py` | 四判据演示（连贯/泛化/可溯源/省电） |
| `ablation_per.py` | 退化验证（注意力 = PER 单次特例） |

## 核心机制

**生成 = 去掩码能量坍缩**：回应区从全 [MASK]（最高能/最无序）出发，
能量网络反复把每个空位填成"让整句最稳定"的字，直到无 [MASK]、能量收敛。

**交互 = PER（预测-误差弛豫）**：每位置向其它位置发预测，按"突触电导"
（经验刻高 = 省电）汇聚预期，用预测误差驱动弛豫到稳定。注意力是它的退化特例。

详见 `kernel/docs/设计v3-M6-最小能量对话模型.md` 与 `设计v3-PER-预测误差弛豫交互.md`，
总结见 `kernel/docs/FE-LLM总纲-我们做出了什么.md`。

## 诚实边界

最大 85 对话 / 158 字 / 1.6M 参数。证明"原理可行"，不证明"能用"。
3 字以上长回应的字序组装仍不稳（非自回归固有难点）。
