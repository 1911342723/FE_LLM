# 自由能核心重做：从“不稳定”到“稳定”的计算本体

> 启动日期：2026-07-19
> 状态：递归稳定化、自动生长、噪声拒绝、低秩生长、真实字符流低秩闭环、多盆地生命周期、
> 真实自然语言→代码漂移与结构自由能慢动力学均完成三种子裁决。
> 旧 `SeqEnergyNet` 与 Transformer 保留为历史基线。

## 1. 为什么重做

旧 `SeqEnergyNet` 使用 Q/K 相容度、softmax 路由、逐层不同参数和 next-token 交叉熵，已经逐渐变成
“用预测误差语言解释的 attention-like 网络”。它保留了可学习突触与可溯源实验价值，但没有把
**自由能下降本身**实现成推理过程。

新核心重新规定四条不可妥协的物理契约：

1. 存在可以直接计算的标量自由能，而不是把 `-logits` 命名为能量；
2. 推理是同一套动力学的反复弛豫，而不是不同网络层的一次前向传播；
3. 每一步自由能不增加，并能按局部收敛程度停止；
4. 位置 `j` 的状态、输出和停止时刻只能依赖 `<=j` 的输入。

## 2. 第一版自由能

对每个位置 `j`，当前 token 形成观测 `a_j`，上一位置的稳定信念状态通过共享生成性转移
`T(z_{j-1})` 给出先验，内部状态为 `z_j`：

```text
F_j(z_j) = po/2 · ||z_j - a_j||²
         + pp/2 · ||z_j - T(z_{j-1})||²
         + pc/2 · ||z_j||²
```

- 第一项：状态不能无代价地背离当前观测；
- 第二项：状态要与过去产生的预测协调；
- 第三项：限制状态复杂度，防止靠无限幅值降低任务损失；
- `po / pp / pc`：第一版固定的正精度，表达系统相信观测、预测和简单性的程度。暂不开放学习，
  防止任务优化器把预测精度压到 0，从而绕过自由能动力学。

内循环反复执行：

```text
z(t+1) = z(t) - eta · dF/dz
```

自由能关于 `z` 是凸二次函数，`eta` 被约束在曲率倒数以内。因此单调下降是实现保证，
不是训练后观察到的偶然现象。

## 3. 严格因果性

模型不使用 Q/K attention。每个位置按顺序读取上一稳定态，使用同一组生成性突触产生先验：

```text
mu_j = T(z_{j-1})
z_j(0) = a_j
z_j(t+1) = z_j(t) - eta · dF_j/dz_j
```

`T` 可非线性，但位置 `j` 弛豫时先验固定，所以局部自由能仍是凸二次。每个位置独立计算下降幅度
和停止掩码；改变未来 token 不会通过全局 line-search 或统一停止时刻泄漏到过去。

## 5. 现在已经证明与尚未证明的事

已经由实现和测试保证：

- 自由能有明确组成和数值轨迹；
- 弛豫使用共享参数，逐步降低自由能；
- 能按位置自适应停止；
- 严格因果，无未来泄漏；
- 共享生成性转移能同时从语言任务与残余自由能目标获得梯度；
- 已接入现有 tokenizer、训练、评估、生成和 checkpoint 路径。

第一项三种子机制裁决（lag-2 ID / lag-3 OOD）进一步证明：

- 完整弛豫 ID acc `100%`，0-step 仅 `13.4%`，弛豫净增益 `+86.6%`：稳定化确实承担上下文计算；
- CE+F 的结构 surprise AUROC `0.992±0.002`，稳定后 residual AUROC 同为 `0.992`；
- CE-only 虽也达到 `100%` 任务准确率，但 residual F 恶化到约 `0.70`，OOD 判别 AUROC 仅 `0.151`；
- 说明“会答”不等于“生成模型能稳定解释经验”，外循环最小化自由能不可省略。

完整报告：[`reports/free_energy_sequence_eval.md`](reports/free_energy_sequence_eval.md)。

第二项三种子“穷则变”裁决进一步完成闭环：

- 阈值只由旧稳定经验 residual-F 的 `99%` 分位校准，不使用技能名或规则标签；
- 已知 lag-2 流误触发率 `0%`，未见 lag-3 流生长触发率 `100%`，新结构压力样本 `98.2%±1.0%`；
- 自动复制最低能旧假设并长出新生成性转移通路，只训练新通路；
- 新通路 lag-3 acc `100%`，旧通路 lag-2 acc `100%→100%`，旧 logits 最大变化严格为 `0`；
- 最低 residual-F 路由：ID→旧通路 `100%`，OOD→新通路 `97.7%±1.4%`，路由后两任务均 `100%`。

这第一次把“穷则变”从人工技能分块改成了自由能状态驱动：现有生成模型解释不了经验才增加容量，
稳定经验继续停留在原有冻结吸引子。完整报告：
[`reports/free_energy_growth_eval.md`](reports/free_energy_growth_eval.md)。

第三项三种子裁决解决“高能噪声导致盲目扩容”：

- 基础数据使用真实 OPUS 英文字符流；结构变化为独立 held-out 句子的反向流；
- 噪声逐 chunk 打乱同一批字符，因此词表、长度和字符边际与结构流完全相同；
- 高能只创建临时通路，临时通路先纯粹最小化生成性 residual-F；
- 结构 held-out 可约率 `35.6%±0.4%`，越过 `30%` 固化门槛，三种子固化 `100%`；
- 噪声 held-out 可约率 `25.5%±0.8%`，三种子固化 `0%`；
- 新通路支付由旧稳定流虚假能量优势校准的 MDL 代价：旧流→旧通路 `95.2%`，
  结构流→新通路 `99.9%`；
- 稳定态通过后冻结生成性转移，只学习专属读出，结构 BPC `5.72→3.57`；
- 旧通路 logits 最大变化仍严格为 `0`。

这把生长条件从“高能”收紧为：**高能 + held-out 可约 + 支付新增结构成本**。完整报告：
[`reports/free_energy_reducibility_eval.md`](reports/free_energy_reducibility_eval.md)。

第四项三种子裁决把生长从“复制完整转移网络”压缩为低维生成性动力学修正：

```text
T_new(z) = T_base(z) + U · tanh(Vz)
```

- 所有秩与完整通路使用相同基础稳定态、逐步相同训练 batch 和相同训练预算；
- 完整新通路新增 `9,360` 参数；rank-8 只新增 `768`，即 `8.2%`；
- rank-2 / rank-4 分别只有 `40.0% / 75.9%`，显示真实容量转折而非任意小模块都能做；
- rank-8 新结构 acc `99.86%±0.05%`，OOD→新通路 `96.5%±1.7%`；
- ID→旧通路 `97.5%±0.9%`，旧通路 logits 最大变化严格为 `0`；
- rank-16 达 `100%`，但参数升至完整通路 `16.4%`，因此 rank-8 是当前最小可用点。

这不是 attention/Transformer LoRA：修正对象是生成性状态转移 `T(z_prev)`，其输出仍进入显式
自由能并通过稳定化动力学决定信念状态。完整报告：
[`reports/free_energy_low_rank_growth_eval.md`](reports/free_energy_low_rank_growth_eval.md)。

第六项把一次性生长扩成“生长→复用→合并→回收”的能量盆地生命周期：

- 真实 OPUS 字符构成上依次出现整句反向与句内字符排序，两者均保持逐句字符边际；
- rank-16 反向结构 residual-F 可约 `31.4%±0.6%`，排序结构可约 `47.4%±0.8%`，
  三种子均连续固化；
- 最低自由能路由为：正常流→基础盆地 `95.3%±0.2%`，反向流→反向盆地 `100%`，
  排序流→排序盆地 `100%`；
- 反向结构再次出现时，现有盆地已能解释，重复生长率 `0%`；
- 显式注入的同盆地持久化冗余快照按 residual-F + MDL 等价性合并，合并率 `100%`；
- 近期“正常+排序”窗口中，反向盆地最低能占比只有约 `2.3%`，低于与路由误差对齐的
  `3%` 回收线，三种子均回收；回收后正常与排序路由均为 `100%`；
- 被回收的反向结构返回后重新产生生长压力 `100%`，基础 logits 最大变化仍严格为 `0`；
- 容量轨迹为 `1→2→3→3→4→3→2`（初始→两次生长→重复复用→冗余注入→合并→回收）。

诚实边界：合并对象是同一盆地的重复快照，尚未证明两个独立训练的相似结构会自动合并；
回收依赖近期窗口，不是无损长期记忆。先导的相邻字符交换在 seed 91、rank-16 下未过固定
`30%` 可约率闸门，已被正确拒绝，正式排序结构仍属于受控机制任务。完整报告：
[`reports/free_energy_lifecycle_eval.md`](reports/free_energy_lifecycle_eval.md)。

第七项不再使用受控字符变换，而是从 OPUS 英文自然语言漂移到 CodeParrot Python 源码；
代码训练和 held-out 来自不同数据分片：

- 漂移窗口中的代码比例从 `0→10%→25%→50%→75%→100%` 增长时，高 residual-F 样本
  比例从 `1.2%→7.3%→17.2%→32.4%→46.9%→62.5%`，三种子全部单调；
- 固定 `25%` 生长压力门槛在代码占比 `50%` 时首次越过，没有领域分类器或关键词触发器；
- `10%` 等字符边际局部污染触发率 `0%`；`50%` 污染 burst 会进入临时 probe，但不会直
  接固化；
- 真实代码 held-out residual-F 可约 `30.9%±0.5%`，逐 chunk 打乱同批代码字符的噪声
  只可约 `26.6%±0.5%`，因此代码固化率 `100%`、噪声固化率 `0%`；
- rank-16 在 seed 101 先导中只可约 `26.2%`，被固定 `30%` 闸门拒绝；rank-32 首次越线，
  正式三种子使用未参与容量选择的 seeds 104–106 与 rank-48；
- MDL 代价覆盖 `90%` 旧稳定流的虚假能量优势，最终旧域→基础盆地 `90.5%±0.1%`，
  代码域→代码盆地 `92.4%±1.1%`；
- held-out 代码 BPC `5.970→3.876`，字符准确率约 `15.5%→27.9%`，基础 logits 最大变化
  仍严格为 `0`；
- rank-48 新增 `12,528` 参数，是完整转移+读出生长 `21,061` 的 `59.5%`，真实跨域的
  容量效率明显弱于受控反向结构，不能据此宣称模型已经高效。

完整报告：[`reports/free_energy_domain_drift_eval.md`](reports/free_energy_domain_drift_eval.md)。

第八项把单窗口的外循环阈值升级为慢时间尺度的显式结构自由能。窗口高能比例为 `p_t`，
结构不稳定慢状态为 `s`：

```text
G_t(s) = po/2 · (s - p_t)²
       + pp/2 · (s - s_{t-1})²
       + pc/2 · s²
```

- 第一项让结构状态响应当前 residual-F 压力；第二项提供结构惯性，拒绝一次 burst 立即改写
  模型结构；第三项让无持续外力的结构复杂度自然衰减；
- 每个窗口内部使用受曲率约束的同一梯度弛豫，结构自由能单调不增率 `100%`；环境切换可以
  对系统做功、抬高下一窗口初始能量，这不与窗口内部耗散矛盾；
- 输入直接重放 seeds 104–106 真实 OPUS→Python 模型测得的 residual-F 压力，不使用人工
  构造的理想概率；
- 原单窗口 `25%` 规则对 `50%` 噪声 burst 触发率 `100%`，慢结构动力学触发率 `0%`；
- 持续渐变漂移在代码比例 `75%` 阶段越过生长势垒，三种子激活率 `100%`；
- 激活势垒 `0.25`、复位势垒 `0.10` 形成迟滞；概念回返后平均 `8.0±0.8` 个稳定窗口
  复位，复位率与迟滞观测率均为 `100%`；
- 这不是额外分类器或 EMA 改名：慢状态由可直接计算的 `G_t` 反复下降得到，报告保存了每个
  在线窗口的初始能量、最终能量和耗散量。

完整报告：[`reports/free_energy_online_drift_eval.md`](reports/free_energy_online_drift_eval.md)。

尚未证明：

- 语言建模能力优于旧 PER 或 Transformer；
- 当前递归生成性转移足以表达复杂语言关系；
- 高残余自由能能在真实语言分布上区分“数据噪声”和“结构容量不足”；
- 当前慢动力学仍是对真实压力时间线的离线重放，尚未与临时通路训练耗时、异步到达和多盆地
  并发竞争组成同一个端到端在线进程；
- 两个独立训练但相似的低能盆地能否只凭能量证据安全合并；
- 低频但重要的旧盆地如何避免被近期窗口误回收，以及是否需要低成本冷存储；
- 低秩生长在机制任务降至完整通路 `8.2%`，真实字符结构漂移的端到端成本为 `19.2%`；
- 最低自由能路由当前需逐通路完整求解，通路数量扩大后的计算代价尚未解决。

因此这一版是**正确物理语义的最小核心**，不是最终架构，也不应提前宣称性能胜出。

## 6. 后续顺序

1. 把慢结构自由能接入端到端异步在线生长，纳入 probe 训练耗时与多盆地并发竞争；
2. 让两个独立学习的相似盆地接受合并裁决，并为回收容量增加冷存储/再激活对照；
3. 将当前逐通路完整求解改成低成本先验筛选，再由自由能做最终裁决；
4. 做完整字符代码 LM，与旧 PER、参数匹配 Transformer 做多种子对照；
5. 最后才扩展到翻译等任务。
