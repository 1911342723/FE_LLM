# FE-LLM 术语表（白话版）

更新日期：2026-06-13

> 把项目里出现的名词用大白话讲清楚，每条：**术语 — 一句话解释 — 在 FE-LLM 里指什么/在哪**。

## 一、主动推理核心概念

- **主动推理（Active Inference）** — 一种决策框架：系统不是被动预测，而是**主动选一个能把"未来的意外"降到最低的行动**。FE-LLM 的大脑就按这个工作（先选动作再生成）。
- **belief / belief 状态（信念状态）** — 模型当前"相信什么"的内部状态：包括它对你意图的理解、已经知道的信息（槽位）、是否在等你补充。代码：`state.py` 的 `BeliefState`，跨轮保存。
- **prediction error（预测误差）** — 模型"原本以为会看到的"与"实际看到的"之间的差。差越大越意外。代码：`surprise.py`。
- **surprise（惊奇）** — 把预测误差加权汇总成的一个分数，分**多通道**：语义、意图、逻辑、不确定、安全。高 surprise 会触发追问/拒答。例：「帮我订票」不确定通道 0.9 → 追问。
- **precision（精度）** — 对某条误差"有多可信"的权重。可信的误差更影响决策，噪声少影响。
- **free energy / expected free energy（自由能 / 期望自由能 EFE）** — 衡量"一个行动好不好"的总分（越低越好），由风险、模糊度、信息价值、代价组成。模型选 EFE 最低的动作。代码：`free_energy.py`。
- **自由能平复** — 通过行动或补充信息，让 surprise/预测误差降下来（例：你追问后用户补全 → surprise 显著下降）。

## 二、控制层与动作

- **controller（控制器）** — 串起整个闭环的"总指挥"：观测→预测→惊奇→更新 belief→选动作→生成→记录→成长。代码：`controller.py` 的 `ActiveInferenceController`。
- **policy（策略）/ 动作选择** — 在固定动作集合里挑一个的过程。代码：`policy.py` + 学习式 `context_policy.py`。
- **动作类型（5 个）**：
  - `answer` 直接回答；
  - `ask_clarification` 信息不足 → 追问；
  - `retrieve` 需要外部/实时信息 → 检索；
  - `refuse` 有风险 → 拒答；
  - `update_memory` 用户表达稳定偏好 → 记下来。
- **pending_clarification** — belief 里的一个标志：表示"上一轮我追问了，正等你补充"。下一轮你补了就能接上。

## 三、理解与槽位（NLU）

- **intent（意图）** — 你这句话想干什么（订票/订酒店/提醒/闲聊…）。
- **slot（槽位）** — 完成一个任务需要的"必填项"。例：订机票需要 `route`(出发到目的) + `date`(日期)；订酒店需要 `city`+`date`。
- **known_slots（已知槽位）** — belief 里已经收集到的槽位值（跨轮记住）。例：你说过「北京到上海」后，`known_slots={route:北京到上海}`。
- **required_slots（必需槽位）** — 当前请求还需要哪些槽位。全部齐了才回答，缺了就追问。
- **NLU（自然语言理解）** — 把你的话解析成"意图 + 槽位"。FE-LLM 有规则版 + 学习版（`nlu/`）。
- **gazetteer（词表/地名表）** — 一份固定的实体清单（如城市名表）。规则槽位抽取靠它；它的局限是认不出表外的新词，所以才上学习式。
- **taxonomy（分类体系）** — 一套"类别的划分标准"。这里指**领域/意图/槽位怎么归类**。例：把任务分成 flight/hotel/reminder… 每类需要哪些槽位，就是一套 taxonomy。"taxonomy 统一"= 让规则版和学习版用**同一套**分类标准（目前两边略有错位，是待办）。

## 四、生成与能量

- **energy decoder（能量解码器）** — 生成文字的方式：逐字挑"让隐状态最接近目标意图向量"的字，使**残余能量递减**。代码：`energy_lm` / `intent_generate.py`。
- **residual energy（残余能量）** — 当前生成状态离"目标意图"还有多远。越生成越小 = 朝意图收敛。例：7.34 → 7.23。
- **intent vector（意图向量）/ belief_mixed** — 把你的话压缩成的一个向量目标；`belief_mixed` 表示生成时混入了控制层的 belief 意图。
- **decode_loss** — 生成模型预测下一个字的平均误差（越低越好）。例：48k 重训把留出集 decode_loss 从 2.18 降到 1.39。
- **predictive coding（预测编码）/ 分层** — 高层预测低层、低层把误差回传的机制；"分层"指多层 latent（z_1..z_L）。FE-LLM 建了引擎但在小任务上没显价值，诚实结案。

## 五、评测/实验术语

- **headroom（提升空间/余量）** — "一个机制还能比基线好多少"的空间。**关键含义**：如果任务太简单、基线已经做满分，机制就没有 headroom（显不出价值）；任务"对"了才有 headroom。例：开放闲聊 belief headroom 弱(0.655)，任务型强(歧义子集 +0.5)。
- **baseline（基线）/ ablation（消融）/ control（对照组）** — 用来比较的"对照版本"。消融=关掉某个部件看掉多少，证明该部件有用。例：关掉 belief（清空）→ 歧义子集从 1.0 掉到 0.5，证明 belief 关键。
- **ambiguous subset（歧义子集）** — **同一句话在不同上下文下应有不同动作**的那批样本。只看句子无法区分，必须靠 belief —— 是检验上下文价值的关键集合。
- **balanced accuracy（平衡准确率）** — 各类别召回率的平均，避免被多数类带偏（类别不均衡时比普通准确率公允）。
- **win rate（胜率）** — 成对比较中 A 优于 B 的比例。例：真实续接 surprise 低于随机句的比例 0.655。
- **held-out（留出集）/ session 切分** — 训练时没见过的数据用来测泛化；按"会话"切分防止同一会话的轮次泄漏到训练。
- **checkpoint** — 训练好的模型权重文件（`.pt`）。

## 六、数据/工程

- **session（会话）** — 一段多轮对话，用 `session_id` 标识；belief 在同一 session 内跨轮累积。
- **LCCC** — 一个真实的中文人类对话语料（开放闲聊）。
- **teacher corpus（教师语料）** — 由本架构作者作为"教师"生成的任务型多轮语料（带动作标签 + belief），用来训练/验证控制层。
- **trace（溯源链路）** — 从观测到行动的完整记录，让"为什么这么回"可查，不靠注意力权重。代码：`trace.py`。

## 七、哲学映射（道易草案，仅注释）

- **道** — 生成秩序：系统沿"最小阻力、最低扰动"的路径自组织（对应"最小干预/无为"的动作偏好）。
- **目** — 注意力：看见相关性、预测下一字。FE-LLM 里注意力只当"目"，决策来自上层显式状态（"道"）。
- **象/卦/爻** — 把局势编码成结构化状态：对应 belief 的结构化（意图 + 槽位 + 精度），而不是单个连续向量。
- **穷则变** — 只有当旧结构解释力到极限才触发结构性成长（对应：重复确认的记忆才进离线再训练）。
