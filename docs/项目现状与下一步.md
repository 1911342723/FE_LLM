## 3. 核心成果：能力矩阵（LoRA 式隔离知识模块 vs 标准 Transformer）

| 能力 | 标准 Transformer | 本模块 | 实验 |
|---|---|---|---|
| 持续学习不遗忘 | ❌ 88%→0% (Δ+9.16) | ✅ Δ0、学会 83% | B/C/D |
| 知识可定位编辑 | ❌ 殃及旁观(特异性 0) | ✅ 特异性 0.77~1.00、旁观 0 影响 | A / Real |
| 自动成长 + 路由 | ❌ 需人工 | ✅ surprise 门控自动长块、import 路由 100% | Router / Real |
| 前向迁移（且不忘） | ✅ 但会忘 | ✅ +34% 且 Δ0 | E |
| **真实库 API 知识上验证** | — | ✅ 不忘 100% / 编辑 +1.00 / 路由 100% | **Real** |

机制同源：**每个知识一个独立低秩 LoRA 块、冻结隔离（+ 侧向连接 + surprise 门控 + import 路由）**。

## 5. 诚实边界（新会话需知）

- 机制验证**规模小**：3~4 个知识、held-out 16~24 实例、字符级 52M 模型——**证机制不证规模**。
- **学习化（能量）路由弱**（44%~94%）；`import`/触发词路由可靠（代码场景天然），是当前推荐方案。
- 隔离类共性代价：**参数随知识线性增长、无前向迁移**（Progressive 部分缓解迁移）。
- LoRA-ISO 模块尚未工程化为独立可复用模块（散在 eval 脚本）。
- 不与通用 LLM 拼世界知识 / 生成博学度（容量纪律）。

## 6. 下一步建议（优先级）

1. **规模化验证**：几十个库/技能 + 更大模型，看隔离/路由/编辑是否保持（最能堵"小规模"质疑）。
2. **把 LoRA-ISO 提炼成一等模块**：从 eval 脚本抽出 `LoRALinear`/隔离容器/Progressive/路由到 `energy_lm/`（如 `lora_growth.py`），配 demo。
3. **写成正式论文/技术报告**：对比 LoRA / Progressive Networks / EWC / MoE，讲清异同与定位。
4. **改进学习化路由**：让模型自身（非 import 关键词）可靠选块（开放问题）。
