# FE-LLM 阶段总结（2026-06-15）

> 这份文档为当前阶段画句号：FE-LLM 是什么、做成了什么（带证据）、什么被诚实证否/封存、
> 边界在哪、距离理想蓝图还差什么。配套的下一步「核心引擎构想」见
> `docs/FE-LLM核心引擎构想.md`。

## 0. 一句话

FE-LLM 是一个**从 0 自建、不依赖预训练底座、可验证、会成长的「主动推理控制架构」**。它的
六要素（能量解释 / 预测误差 / 自由能平复 / 主动推理 / 可溯源 / 自我成长）已端到端打通，并在
**真实任务对话数据**上证明了价值；全量 125 回归测试守护。蓝图里更宏大的「分层预测编码世界
模型」经原则性裁决判为本规模下过度设计、已封存；「生成博学度」按容量纪律主动不竞争。

## 1. 愿景与硬纪律

- 愿景（`道易主动推理方案草案` / `从0自建v2架构设计`）：一个在生成前会"感到 surprise"、形成
  预测误差、再通过对外检索 / 对内推理 / 记忆更新 / 答案生成来平复自由能的认知架构。
- 硬纪律（容量）：**不和通用 LLM 拼世界知识 / 翻译 / 开放问答**——8.9M 从 0 模型物理装不下
  （翻译 word-F1 0.07/0.08 两次证伪）。战场是**机制**：动作选择 / 多轮 surprise / 意图驱动短
  生成 / 可溯源 / 成长。

## 3. belief 在真实数据上的价值地图（B2 系列，本阶段核心新成果）

在真实人标任务对话 **CrossWOZ** 上，用统一口径（同 MLP/split/train，唯一变量=是否加 belief）
测了 belief 三个维度的价值（重点看"领域未明示的跟进句"——belief 唯一能起作用处）：

| 维度 | belief 真实 headroom | 报告 |
|---|---:|---|
| 怎么回应（动作类型） | **无**（−0.02，动作几乎由当前 utterance 决定） | `crosswoz_headroom_eval.md` |
| 谈什么·语境（状态/领域追踪） | **强 +0.19**（0.66→0.86） | `crosswoz_domain_tracking_eval.md` |
| 说什么·内容（回复 领域·槽位 grounding） | **强 +0.15**（0.67→0.83） | `crosswoz_response_content_eval.md` |

**结论**：belief 的用武之地是"谈什么/语境/内容"（状态追踪、领域跟踪、指代、内容 grounding），
不是"怎么回应（动作类型）"。这修正了早期"belief 决定动作"的合成强结论（很大程度是教师语料
的构造特性），又在真实数据上**正面坐实**了机制价值。

落地：B2d/B2e 把这一洞察接回真实 controller——`active_domain` 跨轮追踪，多槽位领域**不必复述
领域**即可主动补全 + 领域切换正确（demo `fe_llm_multidomain_demo.md`）。

## 4. 诚实证否 / 封存（同样是成果）

- **预训练底座线（N1/N2）**：① 注入路线（P1/P1.5/P2/P2c）翻译方向正式集阴性；② N2 句向量
  路线（action 分类 + 领域理解）冻结 Qwen2.5-0.5B 句向量均不优于自建 8.9M 编码器。两面夹证
  封存。底座的强项是**生成/知识**，不在机制——若再用应放生成端。
- **分层预测编码世界模型（v2 核心引擎）**：经**组合泛化裁决**（分层理论上唯一不可替代的好处）：
  未见组合 flat 0.652 ≥ hierarchical 0.637，连理论主场都没赢扁平；加 V2-M1 的 decode_loss
  代价 → 判本规模过度设计，**正式封存**（引擎+测试留档）。证据 `compositional_generalization_eval.md`。

## 5. 距离理想蓝图还差什么

| 蓝图核心 | 状态 |
|---|---|
| 主动推理控制闭环（六要素） | ✅ 已实现 + 真实数据验证（项目真正成果） |
| 分层预测编码世界模型 | ⛔ 已裁决封存（过度设计，非"差一口气"） |
| 生成博学度 / 世界知识 | ❌ 弱，但蓝图主动不竞争（容量纪律） |
| **真正"powerful brain"级核心引擎** | 🟡 **CAPCW 已验证为正面方向**（内容寻址工作空间，见第 5.1 节）；小 d 容量受限 |

**判断（更新）**：FE-LLM 作为"控制架构"已基本完成；核心引擎的"开放问"由 **CAPCW（内容寻址预测编码
工作空间）** 给出了第一个正面答案——它借鉴 Transformer 内核（内容寻址路由），在容量受限区间决定性
胜单向量，并已接回真实 controller 的活文本主动推理闭环。剩余缺口是**容量**（小 d 的绝对天花板，按
容量纪律暂不增 d）与**更难的推理深度**（多跳需 CoT，已验证机制）。

## 5.1 核心引擎 CAPCW 的结论（2026-06-15 大推进，详见 核心引擎构想.md 第 9-23 节）

| CAPCW 判定 | 结果 |
|---|---|
| 内容寻址 slot 工作空间 > 单向量（容量受限绑定） | ✅ PASS（高 K 平均 +0.29） |
| 显式自由能/可溯源形态（PCWorkspace）不丢绑定 | ✅ PASS |
| slot 数=绑定容量 → 穷则变前提 | ✅ PASS |
| 接控制层：价值在内容/状态取回（与 B2 互证） | ✅ PASS |
| 真 in-context 绑定边界（CrossWOZ 可记忆→不需它） | ✅ 边界画清（工作记忆引擎，非世界知识） |
| **序列相邻算子 → induction**（2×2 交互） | ✅ PASS（相邻算子+内容寻址缺一不可） |
| **多跳链式**：潜迭代读 | ⛔ FAIL（写回/深度都不行） |
| **多跳链式**：decode→re-embed+中间监督（潜在 CoT） | ✅ PASS(机制)（+0.30；独立复现"多跳要 CoT"） |
| └ 2×2 析因纠偏（2026-06-16）：多跳主因=**中间监督** | ✅（中间监督主效应 +0.27 ≫ decode→re-embed +0.03；纯涌现 FAIL；§26） |
| └ 自蒸馏中间步（2026-06-16）：中间监督**可自举** | ✅（单跳 teacher 自生成中间步、免 GT 中间标签，达 GT 的 97%、比 final +0.19；§27） |
| └ 朴素自举接活系统（2026-06-16）：校准纠偏 | ⛔ 诚实负（只训单跳→路由弥散 match 0.31<0.5→活系统过度追问；多跳/自蒸馏训练才 sharpen 路由；§28） |
| **接回真实 controller**（工作记忆+surprise→动作+grounded+主动推理 surprise 平复） | ✅ PASS（活文本闭环） |
| **多跳 CoT 接回 controller**（对内多步推理+链式取回+可溯源 CoT trace，2026-06-16） | ✅ PASS（balacc 0.889 / 链尾取回 0.912 / cot−base +0.39 / cot−latent +0.23） |
| **活文本多跳闭环**（复合所有格→链式取回→grounded+CoT trace，2026-06-16） | ✅ PASS（"A的经理的工位"→答；decision/取回/劫持 1.0/1.0/0.0，193 测试全绿） |
| 自我成长（穷则变接活 WM） | 🟡 机制成立、小 WM 不划算（诚实 PARTIAL） |
| 小 d 绑定稳定性 | ✅ 已稳（高方差实为 init 种子 bug，iters=3 最优） |

**一句话**：CAPCW 是 FE-LLM 第一个**有系统实证**的核心引擎方向——内容寻址绑定/induction、接回真实
controller 活文本主动推理闭环都成立；多跳推理的关键被定位为**显式 CoT**，且已接回 controller 的对内
多步推理（链式取回 + 可溯源 CoT trace），并在**活文本路径**跑通复合所有格多跳（"A的经理的工位"→答 +
断链追问→补边→surprise 平复）；唯一硬边界是小 d 容量（按纪律不增 d）。**核心引擎的"开放问"已从"分层
不是答案"推进到"CAPCW 是正面答案、边界清晰"。**

## 6. 方法论沉淀（最值钱的几条，详见 经验.md）

1. **判定机制价值前先确认任务有 headroom，并诊断 headroom 来源**——否则对照只会持平（B2）。
2. **不要为证明一个架构去找"能让它出彩的任务"**（motivated reasoning）；直接测它被理论声称的
   那个好处 + 公平对照（组合泛化裁决 / CAPCW 容量受限区间）。
3. **落地洞察必须用真实多步会话验证**——单测可能"对结果错原因"地通过（B2e 抓到 B2d 被 NLU 架空）。
4. **学习式模块对分布外输入会高置信乱判**；语境类决策让可靠规则信号优先。
5. **容量纪律**：小模型不拼知识/生成博学度，把它用在机制上。
6. **一个机制连试多版才找对**：多跳的写回/深度都是"潜迭代读"的变体（换汤不换药），真正有效的是
   **改变中间表示形态（潜向量→离散符号，即 CoT）**——独立复现"LLM 多跳要 CoT"。
7. **判定阈值别把"机制是否成立(看相对增益)"与"绝对达标(看容量)"塞进一个 AND**（CoT 多跳增益 +0.30
   已证机制成立，绝对偏低是 d=32 容量另一回事）。
8. **归因"训练不稳定"前先排除随机种子未固定**——它能伪装成"高方差"误导整条调参（小 d 绑定稳定性纠偏）。
