# Progressive 式侧向连接：前向迁移 + 仍不忘

解决 LoRA-ISO 的'无前向迁移'代价：每技能新开一列 LoRA，**侧向连接冻结的旧列**——旧列冻结保证不忘，新列借旧列特征拿前向迁移。3 个共享'复制 X'子技能、输出常量不同；少步数(interactions=6)顺序学以暴露学习速度差异。真实 52M 代码模型。

| 技能(学习顺序) | LoRA-ISO 刚学完 acc | Progressive 刚学完 acc |
|---|---:|---:|
| a(第1个) | 6% | 6% |
| b(第2个) | 25% | 38% |
| c(第3个) | 6% | 62% |

- **后学技能前向迁移**（Progressive − ISO，第2/3个均值）：**+34%**
- 不忘：首技能「a」遗忘Δ ISO +0.00 / Progressive +0.00

## 结论

【Progressive 式·前向迁移 + 不忘】少步数(interactions=6)顺序学 3 个共享'复制 X'子技能：
  - 各技能刚学完 acc：ISO [0.06, 0.25, 0.06] / Progressive [0.06, 0.38, 0.62]
  - **后学技能(第2/3个)前向迁移**：Progressive − ISO = +34%（>0 = 侧向借旧列'复制 X'学得更快/好）
  - 不忘：首技能「a」遗忘Δ ISO +0.00 / Progressive +0.00（均≈0，旧列冻结保证）
结论：Progressive 侧向连接在保持'隔离不忘'的同时拿到**前向迁移**（+34%），缓解了 LoRA-ISO'每列从零学、无迁移'的代价。

## 诚实边界

- 前向迁移大小取决于子技能可共享程度与步数预算；本任务共享'复制 X'，少步数下侧向借力更明显。
- Progressive 仍参数随技能增长（每列 + 侧向门）；侧向门 O(列数²) 但极小。
- 侧向连接对旧列做 stop-grad（不改旧列=不忘）；评估旧技能用旧列（不受新列影响）。
- 机制验证（3 技能、held-out 16 实例），不证规模。
