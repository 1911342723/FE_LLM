# 参数效率诊断：学新技能时'参数用对地方了吗'

回答根本质疑「学得多记得少，那不是很多无效参数吗」。同一个学新技能任务（复制规则 `def make_<X>(): return Widget("<X>")`），冻结 backbone，**只训不同的小模块**，比 held-out（没教过的同规则新实例）复制准确率 vs 可训练参数量。

| 臂（可训练模块） | 可训练参数 | lr | held-out acc | held-out loss(bits) |
|---|---:|---:|---:|---:|
| synapse | 786.4K | 0.05 | 0%→**75%** | 0.83→0.42 |
| head | 938.9K | 0.0005 | 0%→**33%** | 0.83→0.43 |
| lora-r8 | 163.4K | 0.002 | 0%→**83%** | 0.83→0.08 |
| lora-r32 | 653.5K | 0.002 | 0%→**92%** | 0.83→0.03 |
| full | 52498.6K | 5e-05 | 0%→**79%** | 0.83→0.11 |

## 结论

【参数效率诊断】学新技能（复制规则），冻结 backbone 只训不同模块，各臂取最佳 lr 后 held-out（24 个新实例）复制准确率：
  - lora-r32: 92%（653K 参数, 最佳 lr=0.002）
  - lora-r8: 83%（163K 参数, 最佳 lr=0.002）
  - full: 79%（52499K 参数, 最佳 lr=5e-05）
  - synapse: 75%（786K 参数, 最佳 lr=0.05）
  - head: 33%（939K 参数, 最佳 lr=0.0005）
最高：lora-r32 92%（653K）。 最省参数达 ≥75% 的是 lora-r8（163K）。
**诚实修正**：synapse 用对 lr（0.05）达 75%——它并非'学不动/参数无效'，之前的低分主要是 lr 未调最优。'学得多记得少'的真因是 stability-plasticity（共享参数学新→忘旧，见 code_forgetting_compare），而非 synapse 参数无效。synapse 的真正局限在'位置门控'对更复杂/冲突映射的表达力、以及容量与 dim 脱钩（=ctx²），而非此复制任务。
对照：full fine-tune 79%——backbone 表达力没问题；'学得多记得少'根在共享→遗忘，非参数无效。

## 诚实边界

- **公平性**：每臂在各自学习率网格里取 held-out 最佳（表中 lr 列为最佳值）——给每个臂都最好的机会，对照展示的是**模块归纳偏置**差异（位置门控 vs 内容相关 vs 低秩），非超参不公平。
- 任务为单条复制规则的机制诊断（held-out 24 个新实例，acc 粒度 = 1/24），证'参数效率/载体选择'，不证规模。
- LoRA 仅作用于 W_pred + head；可进一步探索作用位置（to_q/to_k/ffn）与秩。
- 本实验只看'学新能力'；'可隔离不遗忘'见 code_forgetting_compare，二者结合指向'内容相关 + 低秩 + 可隔离 + 可溯源'的新知识模块。
