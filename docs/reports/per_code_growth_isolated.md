# 真成长不覆盖：加容量+隔离 vs 共享覆盖

3 条复制规则顺序学习，synapse-only、冻结 backbone。

| 首技能「str」held-out loss | 刚学完 | 学完全部3技能后 | 遗忘Δ |
|---|---:|---:|---:|
| ISO 加容量+隔离 | 0.80 | 0.80 | **+0.00（恒定）** |
| SHARED 共享覆盖 | 0.81 | 1.34 | +0.53（遗忘） |

末态各技能 loss：ISO [0.8, 0.91, 0.64] / SHARED [1.34, 1.24, 0.5]（SHARED 早学的 str 被覆盖到 1.34）。

## 结论

✅ 隔离=真成长不覆盖：首技能「str」held-out loss 在后续学习中 ISO Δ=+0.00（恒定，冻结快照学后续不改它）vs SHARED Δ=+0.53（0.81→1.34，被覆盖）。 诚实 nuance：ISO 每技能从底座独立学=不遗忘但**无前向迁移**（后学技能较弱）；SHARED 有迁移但灾难遗忘早技能。理想解=Progressive 式冻旧块+侧向连接（迁移且不忘）。代价：参数线性增长。

诚实代价：参数随技能线性增长（每技能 +786k，可低秩压缩）；部署需路由选用哪块（隔离保证不遗忘，路由是另一可解问题）。
