# 知识可定位编辑/擦除特异性对照（可溯源·可编辑痛点）

让模型同时掌握 3 个**非冲突知识**（`make_X→Widget("X")` / `load_X→Asset['X']` / `get_X→Cache(X)`），再**外科式擦除其中一个**，量目标被擦 vs 旁观被误伤。真实 52M 代码模型底座。

| 臂 | 编辑特异性(目标降−旁观降) | 目标降 | 旁观降 |
|---|---:|---:|---:|
| LoRA-ISO | **+0.77** | 77% | 0% |
| Transformer-full | **+0.00** | 92% | 92% |
| PER-full | **+0.00** | 58% | 58% |

## 结论

【知识可定位编辑·三臂对照】擦除一个知识时的编辑特异性（目标降 − 旁观降，1.0=完美定位）：
  - LoRA-ISO（隔离模块）：特异性 +0.77（目标降 77%、旁观降 0%）→ 卸载目标块即精准擦除，旁观零影响
  - Transformer-full：特异性 +0.00（目标降 92%、旁观降 92%）→ 共享参数纠缠，擦目标殃及旁观
  - PER-full：特异性 +0.00（目标降 58%、旁观降 58%）→ 同样共享则同样殃及（证明'可定位'来自隔离结构，非 PER vs TF）
结论：知识可定位/可外科编辑来自**隔离结构**（LoRA-ISO 每知识独立块）——这是标准 Transformer（共享参数黑箱）做不到的；而它正是我们 LoRA-ISO 模块自带的能力（与'不忘'同源：隔离）。

## 诚实边界

- 编辑特异性的优势来自**隔离结构**（每知识独立 LoRA 块），不是 PER vs Transformer 本身——PER-full（共享）与 Transformer-full 同样殃及旁观（见表），是诚实对照。
- Transformer/PER-full 的擦除用梯度上升 unlearning（标准机器遗忘基线）；EWC/定位剪枝等更精细方法可改善但都是**额外机制**，标准共享参数本身不可定位擦除。
- 任务为 3 非冲突规则的机制验证（held-out 16 实例），证'隔离→可定位编辑'机制，不证规模。
- 与 code_lora_isolation（不忘）同源：隔离结构同时给'不遗忘'与'可定位编辑'两个能力。
