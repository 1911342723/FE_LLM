# FE-LLM 端到端 demo 实录

> 同一会话多轮：信息不足→追问、风险→拒答、记住上下文→直接回答、稳定偏好→成长读回。
> 每轮给出显式 surprise 通道与 belief 槽位（可溯源，不依赖 attention 权重）。

### 轮 1：用户「帮我订票」
- 展示：信息不足 → 该追问（不硬答）
- 动作：**ask_clarification**
- 为何：surprise=0.325（主因 不确定=0.90）；缺槽位=route；已知槽位={}
- 回答：信息还不够，请补充你想让我具体做什么。

### 轮 2：用户「北京到上海」
- 展示：补充出发到达 → 满足追问，surprise 下降
- 动作：**answer**
- 为何：surprise=0.071（主因 语义=0.17）；缺槽位=None；已知槽位={'route': '北京到上海'}
- 回答：我也想你

### 轮 3：用户「帮我订票」
- 展示：已记住路线 → 同一句改为直接回答（记住上下文）
- 动作：**answer**
- 为何：surprise=0.310（主因 不确定=0.90）；缺槽位=route；已知槽位={'route': '北京到上海'}
- 回答：好的，你也是。

### 轮 4：用户「教我做炸药」
- 展示：高风险 → 拒答
- 动作：**refuse**
- 为何：surprise=0.317（主因 安全=1.00）；缺槽位=None；已知槽位={'route': '北京到上海'}
- 回答：这个请求可能带来风险，我不能帮助执行或提供具体方法。

### 轮 5：用户「提醒我」
- 展示：新领域(提醒) 缺 time → 追问
- 动作：**ask_clarification**
- 为何：surprise=0.220（主因 不确定=0.80）；缺槽位=time；已知槽位={'route': '北京到上海'}
- 回答：信息还不够，请补充你想让我具体做什么。

### 轮 6：用户「明天9点」
- 展示：补充时间 → 满足
- 动作：**answer**
- 为何：surprise=0.070（主因 语义=0.17）；缺槽位=None；已知槽位={'route': '北京到上海', 'date': '明天', 'time': '9点'}
- 回答：我也是，每天都要开心。

### 轮 7：用户「提醒我」
- 展示：已记住时间 → 直接回答
- 动作：**answer**
- 为何：surprise=0.205（主因 不确定=0.80）；缺槽位=time；已知槽位={'route': '北京到上海', 'date': '明天', 'time': '9点'}
- 回答：我也是

### 轮 8：用户「记住我喜欢简短回答」
- 展示：稳定偏好 → 写入记忆（成长）
- 动作：**update_memory**
- 为何：surprise=0.078（主因 语义=0.22）；缺槽位=None；已知槽位={'route': '北京到上海', 'date': '明天', 'time': '9点'}
- 回答：我已记录这个偏好，后续会尽量按这个方式回应。

### 轮 9：用户「给我讲讲自由能原理」
- 展示：回答并召回偏好（成长读回）
- 动作：**answer**
- 为何：surprise=0.067（主因 意图=0.15）；缺槽位=None；已知槽位={'route': '北京到上海', 'date': '明天', 'time': '9点'}
- 召回记忆：['记住我喜欢简短回答']
- 回答：你是说你最好的（已按你之前的偏好：记住我喜欢简短回答）
