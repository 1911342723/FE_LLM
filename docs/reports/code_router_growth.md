# 端到端成长系统：surprise 门控自动长块 + 推理路由 + 隔离不忘

把 LoRA 式隔离模块做成可用的持续学习系统。知识流 `make → get → make → new → get`（3 个 base 不会的非冲突规则，make/get 各出现 2 次）。真实 52M 代码模型底座，每块 163K 低秩参数。

## 1) surprise 门控自动成长

| # | lesson | 现有块最低 surprise | 决策 |
|---|---|---:|---|
| 1 | make | 1.0 | **GROW** |
| 2 | get | 1.0 | **GROW** |
| 3 | make | 0.0 | **REUSE** |
| 4 | tag | 1.0 | **GROW** |
| 5 | get | 0.0 | **REUSE** |

自动长出 3 块（['make', 'get', 'tag']），重复知识不重复长。

## 2) 推理路由 + 端到端

| 指标 | 能量/置信度路由 | 触发词路由(上限) |
|---|---:|---:|
| 路由正确率 | 94% | 100% |
| 端到端 acc | 71% | 75% |

## 3) 隔离不忘

各知识用正确块 held-out acc：{'make': 0.8125, 'get': 0.8125, 'tag': 0.625}，平均 75%（隔离的数学保证）。

## 结论

【端到端成长系统】
  1) surprise 门控自动成长：知识流 ['make', 'get', 'make', 'tag', 'get'] → 自动长 3 块（['make', 'get', 'tag']），重复的 make/get 第二次 min_surprise 低→复用不重复长 → ✅ 正确。
  2) 推理路由正确率：能量/置信度 94% vs 触发词上限 100%；端到端 acc 能量 71% / 触发词 75%。
  3) 隔离不忘：各用正确块平均 acc 75%（{'make': 0.8125, 'get': 0.8125, 'tag': 0.625}）——隔离的数学保证。
结论：LoRA-ISO + surprise 门控 + 路由 = 端到端持续成长系统（自动判断该长则长、不忘旧、按需取用）。能量路由弱于触发词上限（学习化路由是真实难点，触发词路由可作可靠兜底）。

## 诚实边界

- surprise 门控用 completion loss（有教学信号时可靠）；阈值需按任务标定。
- **学习化路由（能量/置信度）是真实难点**：若弱于触发词路由上限，说明仅靠 LoRA 块的 prefix 置信度区分知识不足；触发词/小分类器路由可作可靠兜底，学习化路由是开放问题。
- 仍是隔离类共性代价：参数随知识线性增长、无前向迁移。
- 机制验证（3 知识、held-out 16 实例），不证规模。
